# Brio — Project Reference

## What is Brio?

Brio is a music encyclopedia built on the belief that every song has a world behind it. It is a search-first platform where anyone can ask anything about music — a song, an artist, a feeling, a cultural moment, a place, a genre — and receive a rich, human, musicologically-grounded encyclopedia entry generated by Claude and permanently cached in a database.

The user types "Who is Googoosh" or "Sad songs from Korea" or "Why did disco happen" and gets back a full scholarly entry with a timeline, story, sociopolitical context, genre history, similar works from across the world, influences, and contemporaries. That entry is saved and the next person who asks the same thing gets the cached version instantly.

## Tech Stack

- **Framework**: Next.js 14 (App Router, TypeScript)
- **Database**: Supabase (PostgreSQL via @supabase/supabase-js)
- **AI**: Anthropic Claude API — model `claude-sonnet-4-6`
- **Styling**: Tailwind CSS v3
- **Fonts**: Cormorant Garamond (serif headings), Jost (sans-serif UI)

```
lib/
  supabase.ts               — Exports supabase (anon client) and supabaseAdmin (service role)

```

## Tone Guidelines

Brio writes like a brilliant friend with a PhD in musicology who loves sharing what they know. The voice is:

- Warm, clear, matter-of-fact. Never condescending. Never dry.
- Musicologically rigorous: real theory, real history, real people and places.
- Universally inclusive: the Western European classical canon is one tradition among thousands.
- Honest: never invents quotes, events, or unearned significance.
- Surprising: every entry opens at least one door the reader did not know existed.

## System Prompt

The full Claude system prompt lives at **`lib/systemPrompt.ts`** as the exported constant `SYSTEM_PROMPT`. It is imported in `app/api/search/route.ts` and passed as the `system` parameter to every Claude API call.

The prompt instructs Claude to:
- Detect intent from any natural language query
- Assign one of five subject_type values: song, artist, genre, country, era
- Return structured JSON with: title, timeline (6-10 items), story (3 paragraphs), sociopolitical_context (2 paragraphs), genre_timeline (5 items), similar_across_regions (3-5 items), why_it_matters, influences (up to 4), influenced (up to 4), contemporaries (3), accuracy_note

## Database Schema

```sql
entries (
  id            uuid primary key default gen_random_uuid(),
  slug          text unique not null,
  content       jsonb not null,
  subject_type  text,
  region        text,
  decade        text,
  country       text,
  mood_tags     text[],
  genre_tags    text[],
  title         text,
  created_at    timestamptz default now()
)
```

Run `create-entries-table.sql` in the Supabase SQL editor to create this table.

## Entry Generation Flow

1. User types a query on the homepage
2. Homepage converts to slug and navigates to `/entry/[slug]?q=...`
3. Server component (`app/entry/[slug]/page.tsx`) checks Supabase for existing entry
4. If found: renders `EntryDisplay` immediately with cached data
5. If not found: renders `GenerateEntry` client component which calls `POST /api/search`
6. Search API calls Claude with the full system prompt, parses JSON, saves to Supabase
7. `GenerateEntry` receives the response and renders `EntryDisplay`
8. Future requests for the same slug are served instantly from Supabase

## Environment Variables

```
NEXT_PUBLIC_SUPABASE_URL       — Supabase project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY  — Supabase publishable/anon key (safe for browser)
SUPABASE_SERVICE_ROLE_KEY      — Supabase secret key (server only, bypasses RLS)
ANTHROPIC_API_KEY              — Anthropic API key (server only)
```

## Running the Project

```bash
npm run dev
```

Then open http://localhost:3000.

Before the first run, make sure you have:
1. Created `.env.local` with all four environment variables
2. Run `create-entries-table.sql` in the Supabase SQL editor
